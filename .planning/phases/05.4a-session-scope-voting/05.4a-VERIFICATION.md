### Poirot findings (deferred from Wave 3 + Unit 5.1 QA)

1. **ES2022 `Error.cause` bump** — `voteRepository.ts` uses a manually-declared `cause` field because the project's TS lib target is ES2020. When the lib target bumps to ES2022, remove the manual field and rely on native `Error.cause`.
2. **voteRepository.test.ts mock shape drift** — delete chain mock returns `{ error }` instead of `{ data: null, error }`. Test-only, but brittle if a `.select()` is added.
3. **T-054A-002 double-invocation** — one-assertion-per-test violation. Cosmetic.
4. **Test ID naming convention** — T-054A-* IDs embedded in test names rather than `// comment` above `it()`. Convention drift.
5. **`require('react')` inside vi.mock factories** — CJS/ESM interop hazard. Works under current Vitest config but fragile under strict ESM mode.
6. **DotVoteControls invariant comment** — Add `// NOTE:` documenting that `DotVoteControls` consumers must wrap in `DotVotingProvider` or the internal `useDotVotingContext()` throws.
7. **T-054A-157 exact-count assertion** — `getAllByTestId` with `length >= 1` should be `toHaveLength(1)`.
8. **MatrixFullScreenView provider teardown race** — theoretical structural risk when `votingManager` and `activeSessionId` clear in different render cycles.
9. **T-054A-155 redundant with T-054A-153** — test bloat; remove or strengthen to assert a distinct property.

### Retro lessons

1. **Phase sizing was too aggressive.** "Large" downsized to 100-150k was unrealistic for a 5-unit phase with multi-client realtime complexity. Actual burn was ~1.3M which matches what a true Large phase with full ceremony would cost.
2. **Test spec assumptions must be verified against the real render topology.** Cal's original Unit 5 assumed DotVoteControls could render inline in MatrixFullScreenView. The real render path (DesignMatrix child) required Unit 5.1 addendum. Root cause: Cal did not read DesignMatrix.tsx during initial ADR production.
3. **Unfalsifiable integration tests.** Wave 3 initial integration tests used `if (found) expect(...) else expect(wrapper).toBeDefined()` fallbacks that could never fail. This is exactly the test anti-pattern that masked the missing render site. Pipeline lesson for future phases: Roz must explicitly check for fallback branches in integration test review.
4. **Blind review (Poirot) catches what test-driven review (Roz) misses.** Across Waves 2, 3, and Unit 5.1, Poirot found real issues Roz's tests-passing verdict missed: polling dark mode, DELETE event heuristic, removeVote stale ref, resubscribe race, doubled manager instances, Rules of Hooks violation in mock, and the sr-only render site itself. This validates the mandatory Poirot gate.
5. **Process deviation: Colby committed directly in Waves 1 and 3 instead of routing through Ellis.** Noted for retro.
