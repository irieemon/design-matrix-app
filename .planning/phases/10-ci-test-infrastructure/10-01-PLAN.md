**For the 2 Playwright E2E test files** (`invite-flow.spec.ts` and `project-realtime-matrix.spec.ts`):

Replace each `test.skip('Test name', ...)` with `test('Test name', ...)` but add a conditional skip at the top of the describe block or at file level. Use Playwright's built-in `test.skip(condition, reason)` syntax:

At the top of each test file, after imports and env var declarations, add:
```typescript
const SKIP_LIVE = !process.env['CI_SUPABASE'];
```

Then replace each `test.skip('T-XXX: description', async ...` with:
```typescript
test('T-XXX: description', async ({ browser }) => {
  test.skip(SKIP_LIVE, 'Requires live Supabase (set CI_SUPABASE=true)');
  // ... rest of test body unchanged
});
```

This pattern means: in CI (where CI_SUPABASE=true), the condition is false so tests run. In local dev (no CI_SUPABASE), the condition is true so tests skip with the reason message.

**For the Vitest integration test file** (`phase05.3-migrations.integration.test.ts`):

The file already has a `prerequisitesMet` boolean (line 63-70). Replace each `it.skip(...)` with `it(...)` but add a conditional skip. Use Vitest's `it.skipIf` or add an early return:

Replace each `it.skip('description', async () => {` with:
```typescript
it('description', async () => {
  if (!process.env['CI_SUPABASE'] || !prerequisitesMet) {
    console.log('Skipping: requires CI_SUPABASE=true and all E2E_COLLAB_* env vars');
    return;
  }
  // ... rest of test body unchanged
});
```

This is a simple early-return pattern that keeps the test registered (visible in test output) but skips the body when prerequisites aren't met.

Do NOT change any test logic, assertions, or env var names. Only change the skip mechanism.
