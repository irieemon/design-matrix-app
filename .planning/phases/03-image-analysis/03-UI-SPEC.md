| Token | Value | Usage |

## Interaction States

### Tab Navigation

The AIIdeaModal gains two tabs: "AI Generate" (existing flow) and "Image" (new flow).

| State | Visual |
|-------|--------|
| Tab idle | `text-graphite-500`, no underline |
| Tab hover | `text-graphite-700` |
| Tab active | `text-graphite-900`, `border-b-2 border-sapphire-500` |

Tab implementation: horizontal row of two text buttons at top of modal content area, below the existing header. Use `useState` for `activeTab: 'generate' | 'image'`. Default tab remains "AI Generate" to preserve existing behavior.

### Drop Zone (Image Upload Area)

| State | Visual |
|-------|--------|
| Idle | Dashed `border-2 border-graphite-200` on `bg-canvas-secondary`, centered Upload icon (lucide `ImagePlus`, 32px, `text-graphite-400`), body text below |
| Drag hover | `border-sapphire-400 bg-sapphire-50`, icon color `text-sapphire-500`, pulsing ring animation |
| File selected (preview) | Drop zone replaced by image thumbnail (120x120px, `object-cover`, `rounded-md`), filename + file size text, remove button (X icon, `variant="ghost"`) |
| Disabled (during upload/analysis) | `opacity-50 pointer-events-none` |

Drop zone accepts: `image/jpeg`, `image/png`, `image/webp`, `image/gif`. Max file size before resize: 20MB. After client-side resize: max 4MB.

File input: hidden `<input type="file" accept="image/*">`, triggered by clicking the drop zone or a "Browse files" text link inside the drop zone.

Drag-and-drop: use native HTML5 drag events (`onDragOver`, `onDragEnter`, `onDragLeave`, `onDrop`). No library needed.

### Upload + Analysis Progress

| State | Visual |
|-------|--------|
| Resizing | Inline text "Resizing image..." with `LoadingSpinner` (sm) |
| Uploading | Progress bar (sapphire-500 fill on graphite-100 track, `rounded-full`, height 4px), percentage text below |
| Analyzing | Inline text "Analyzing image with AI..." with `LoadingSpinner` (sm), shimmer animation on preview card |
| Complete | Progress elements replaced by analysis result card |
| Error | Red alert box: `bg-error-50 border border-error-200 rounded-md p-4`, AlertCircle icon (lucide), error message, "Try Again" button (variant `secondary`) |

Progress sequence: Resize -> Upload -> Analyze. Each step shows its label. The progress bar is only visible during the "Uploading" step (the only step with measurable progress via Supabase upload).

### Analysis Result Preview

Mirrors the existing "Generated Idea Preview" card in the AI Generate tab:

| Element | Spec |
|---------|------|
| Container | `border border-hairline-default rounded-lg p-4 bg-canvas-secondary` |
| Header | "Image Analysis" heading with `Eye` icon (lucide), `text-lg font-semibold text-graphite-900` |
| Image thumbnail | 80x80px rounded preview of the uploaded image, float left or inline-start |
| Subject | Label "Subject:" in `text-sm font-medium text-graphite-700`, value in `text-graphite-900` |
| Visual elements | Label "Visual Elements:", comma-separated list in `text-sm text-graphite-700` |
| Extracted text | Label "Extracted Text:" (only if `textContent` is non-empty), value in `text-sm text-graphite-700`, max 3 lines with "Show more" toggle |
| Insights | Label "Insights:", bulleted list in `text-sm text-graphite-700` |
| Relevance score | Inline badge showing relevance score (0-100), color: `lux-badge-primary` if >= 70, `lux-badge-warning` if 40-69, `lux-badge-secondary` if < 40 |
| Re-analyze button | "Re-analyze" with RefreshCw icon, variant `ghost`, size `sm` -- triggers new analysis call with same image |

### Action Buttons (Bottom)

Same layout as existing AI Generate tab:

| Button | Variant | Position | Condition |
|--------|---------|----------|-----------|
| Cancel | `secondary` | Left (flex, full width) | Always enabled |
| Create Idea from Analysis | `sapphire` | Right (flex, full width) | Enabled only when analysis result exists |

### Destructive Actions

| Action | Confirmation Approach |
|--------|----------------------|
| Remove uploaded image (before analysis) | No confirmation -- instant removal with X button. Lightweight action, user can re-upload immediately. |
| Remove uploaded image (after analysis) | No confirmation -- analysis result is not persisted until "Create Idea" is clicked. User loses nothing permanent. |

No destructive actions require a confirmation modal in this phase. The image is only persisted in Supabase Storage after upload, but the idea is not created until the user explicitly clicks "Create Idea from Analysis". Closing the modal without creating the idea is a soft discard.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Drop zone keyboard access | Drop zone is a `<button>` element (or contains a focusable trigger) so users can activate file picker with Enter/Space |
| Drop zone ARIA | `role="button"`, `aria-label="Upload an image for AI analysis"` |
| File input | Hidden `<input type="file">` triggered programmatically, `aria-label="Choose image file"` |
| Progress announcements | `aria-live="polite"` region wrapping the status text ("Resizing...", "Uploading...", "Analyzing...") |
| Image preview alt text | `alt="Preview of uploaded image"` on thumbnail |
| Tab navigation | Tabs use `role="tablist"`, each tab uses `role="tab"` with `aria-selected`, tab panels use `role="tabpanel"` |
| Focus management | When switching tabs, focus moves to the tab panel content area |
| Error announcements | Error messages in `aria-live="assertive"` region |

## Responsive Behavior

| Viewport | Adjustment |
|----------|------------|
| Desktop (>= 768px) | Modal at `max-w-2xl` (existing), drop zone 160px height, image preview inline with analysis text |
| Mobile (< 768px) | Drop zone fills modal width, image preview stacks above analysis text (column layout), buttons stack vertically at `sm` sizes |

The existing `AIIdeaModal` uses `max-w-2xl` which works for both viewports. No new breakpoint-specific layout is needed beyond stacking the analysis result card content on narrow screens.
