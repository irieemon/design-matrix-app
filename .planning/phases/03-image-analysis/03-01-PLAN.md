Create `src/lib/imageResize.ts` exporting:

```typescript
export interface ResizeOptions {
  maxEdgePx?: number  // default 2048
  quality?: number    // default 0.85
}

export async function resizeImageToFile(
  file: File,
  options: ResizeOptions = {}
): Promise<File>
```

Implementation MUST:
1. Use `URL.createObjectURL(file)` + `new Image()` to load the source
2. Compute `longestEdge = Math.max(img.width, img.height)`
3. If `longestEdge <= maxEdgePx`, call `URL.revokeObjectURL` and resolve with the original `file` unchanged
4. Otherwise compute `scale = maxEdgePx / longestEdge`, set canvas dimensions to `Math.round(width * scale)` and `Math.round(height * scale)`
5. `ctx.drawImage(img, 0, 0, canvas.width, canvas.height)`
6. `canvas.toBlob(cb, 'image/jpeg', quality)` — on null blob reject with `new Error('Canvas resize failed')`
7. Wrap blob in a new `File([blob], newName, { type: 'image/jpeg' })` where `newName = file.name.replace(/\.[^.]+$/, '.jpg')`
8. On `img.onerror` revoke object URL and reject with `new Error('Image load failed')`
9. Defaults: `maxEdgePx = 2048`, `quality = 0.85`

Since JSDOM does not implement Canvas, mock `HTMLCanvasElement.prototype.getContext` to return a fake context with a no-op `drawImage`, and mock `HTMLCanvasElement.prototype.toBlob` to synchronously invoke the callback with `new Blob(['resized'], { type: 'image/jpeg' })`. Also stub the global `Image` constructor to dispatch `onload` with configurable `width`/`height` on a microtask.
