# Phase 05.4b: Project-Scope Realtime Matrix тАФ Context

## Goal

Multiple users viewing the same project matrix see each other's presence via avatar stack, see each other's cursors (Figma-style, throttled), see each other's drag operations as soft-locked cards, and see idea position updates in real-time when another user drops a card in a new location. Ideas created on the matrix view (not in a brainstorm session) sync across clients in real-time.

## Scope

**In scope:**
- `ProjectPresenceStack` component тАФ avatar stack for users viewing the same project (mirrors SessionPresenceStack pattern from 05.4a)
- Live cursors layer тАФ Figma-style throttled cursor tracking on the matrix
- Soft drag lock тАФ when user A drags a card, user B sees it visually locked until drop
- Matrix position broadcast тАФ idea drops propagate to all connected users via `@dnd-kit` `DragEndEvent` integration
- Non-session idea sync тАФ ideas created on the matrix view (outside brainstorm sessions) sync via postgres_changes
- Reconnecting badge тАФ 1.5s disconnect threshold, "Back online. Synced." toast on recovery

**Out of scope:**
- Session-scope anything (already shipped in 05.4a)
- Voting controls (already shipped in 05.4a)
- New realtime infrastructure тАФ `ScopedRealtimeManager` already supports project scope via D-09 extension contract (test-verified by T-054A-012/012b)

- **Drag infrastructure:** `@dnd-kit/core` with `DndContext`, `PointerSensor`, `DragEndEvent`
