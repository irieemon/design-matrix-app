## Retro Lessons (additive to 05.4a)

1. **Colby truncation is a persistent pattern** — 3 of 4 Colby invocations truncated at ~130-140k tokens / ~80 tool uses. Eva's post-truncation verification-and-commit protocol works but adds overhead. Consider splitting large waves into smaller Colby invocations in future phases.
2. **Vacuous tests recurred despite retro lesson** — 5 Wave 3 integration tests used `.toBeDefined()` as sole assertions, the same class of issue 05.4a retro explicitly warned against. Root cause: DndKit doesn't expose easy synthetic drag events for testing, so Colby took the path of least resistance. `test.todo` is the honest alternative.
3. **Cal's budget estimate was accurate** — 1.35M estimated, 1.249M actual. Honest Large sizing (retro lesson 4 from 05.4a) proved correct.
4. **ScopedRealtimeManager handler unsubscribe API was correctly deferred** — batching the D-09 exception into Wave 2 (where useLiveCursors also needed it) was the right call. All existing consumers updated atomically.
5. **CSS transition property mismatch** (LiveCursorsLayer `transform` vs `left/top`) — animation specs need explicit property verification in test spec, not just class-name checks.
