## What was wrong (root cause stack)

The transcribe-audio 500 turned out to be **three** stacked code bugs masked by **one** revoked secret, plus **one** more code bug surfaced after the first three were fixed:

| # | Layer | Bug | Symptom | Fix |
|---|-------|-----|---------|-----|
| 1 | server | `fetch(audioUrl)` failed because `audioUrl` was a Supabase Storage path, not a URL | `TypeError: Failed to parse URL from projects/...webm` | `08b03f5` — added `resolveAudioUrl()` helper that mints a 1-hour signed URL via service-role client when input isn't `http(s)://` |
| 2 | env | `OPENAI_API_KEY` in Vercel prod was revoked/invalid | `Incorrect API key provided: sk-proj-...hioA` from Whisper | manual rotation in Vercel dashboard |
| 3 | client | `FileService.uploadFile` used singleton supabase client → internal `getSession()` deadlocks (anti-pattern #2 / supabase-js #873) | Modal stuck on "Uploading" forever; HAR shows zero POSTs to storage endpoint | `c8c8815` — uploadFile and cleanupFailedUpload now use `createAuthenticatedClientFromLocalStorage()` |
| 4 | client | `MediaRecorder.start()` called with no timeslice → `ondataavailable` race produces header-only blob | "you" hallucination from Whisper on ~2.7 KB webms | `4226a62` — `recorder.start(250)` so chunks accumulate during recording |
| 5 | server | Summarization step at line 122 used `getModel()` → `getGateway()` → required `AI_GATEWAY_API_KEY` not provisioned | iPhone: `AI_GATEWAY_API_KEY environment variable is not set` | `5e41531` — reuse `@ai-sdk/openai` provider already in this file, call `openai('gpt-4o-mini')` directly |
