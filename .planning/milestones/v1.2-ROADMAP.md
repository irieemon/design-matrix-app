## Overview

v1.2 took Prioritas from "feature-complete but brittle in CI" to "production-hardened with a working CI test matrix against live Supabase." Three phases closed deferred operational gaps from v1.0/v1.1 (Resend domain, CSRF race, AI Gateway), fixed the root-cause BUG-01 realtime dual-subscription race (removing the D-34 workaround), and stood up an ephemeral-Supabase GitHub Actions workflow so integration tests finally run against real infrastructure instead of being gated as `test.skip`.

The milestone closed with a 6-pipeline E2E triage arc that fixed four layers of access-control bugs revealed once the CI suite started actually executing. The arc ended at CI Integration Tests 3/10 green — with the remaining 7 failures **explicitly re-scoped into v1.3 — E2E Realtime Hardening** (React render lifecycle, invite-flow navigation, pre-existing `accept_invitation` RPC bug). That re-scope was deliberate: the retro-lesson "onion-peel strategy change signal" fired after 3+ consecutive pipelines with unchanged pass count, and the remaining bug class (React realtime) is mismatched with per-test Eva orchestration against a 5-minute CI loop.

**Net effect:** Access-control chain (URL routing → user_profiles schema → token extraction → collaborator validation) is bulletproof. CI test matrix exists and runs. Everything else that remains is ring-fenced in v1.3 with local repro as prerequisite.
